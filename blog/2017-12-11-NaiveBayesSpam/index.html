# Finds all unique types with count in a dataframe per column
def findtypes(df,series=False):
    d = {}
    if series == False:
        for item in df:
            clas = re.findall('\'(.*?)\'',(str(pd.Series(list(map(type,df[item]))).unique())))
            vals = list(pd.Series(list(map(type,df[item]))).value_counts())
            d[item] = ['{}: {}'.format(clas,vals) for clas,vals in zip(clas,vals)]
        return d
    else:
        clas = re.findall('\'(.*?)\'',(str(pd.Series(list(map(type,df))).unique())))
        vals = list(pd.Series(list(map(type,df))).value_counts())
        return ['{}: {}'.format(clas,vals) for clas,vals in zip(clas,vals)]

We're almost ready to build our machine learning model, but first we should understand Bayes' Theorem and Vectorization.

Bayes' Theorem¶

Bayes theorem is a method to calculate conditional probability.
We read $P(A|B)$ as the probability of A given B. Which means if B were to occur, what is the probability of A occuring. $$P(A|B) = \frac{P(B|A) \times P(A)}{P(B)}$$

To make sense of the formula let's go through an email spam example. Here we assign the probability of spam and not spam to be equal at 50%. Where A' represents not A, the complement of A.

Step 1 - Prior - Find probability of our dependent variable
$P(A) = P(spam) = 0.5$
$P(A') = P(not \space spam) = 0.5$

Step 2 - Joint - Find probability of our independent variables given our two scenarios
For simplicity we'll assume only 3 words are used in emails: "prize", "locksmith" and "morning". We'll assign probabilities to them. We'll say emails with prize have a high chance (70% chance) of being spam.
$P(prize|spam) = 0.7$
$P(locksmith|spam) = 0.1$
$P(morning|spam) = 0.2$

$P(prize|not \space spam) = 0.1$
$P(locksmith|not \space spam) = 0.5$
$P(morning|not \space spam) = 0.4$

Step 3 - Normaliser - Find probability of our independent variable
So if we now have an email containing "morning prize" how should we classify this email?
$P(morning \space prize) = P(morning \space prize|spam) + P(morning \space prize|not \space spam)$

We can calculate this by adding the probability of "morning prize" given it is spam and the probability of "morning prize" given it is not spam.
$P(morning \space prize|spam) = P(morning|spam) \times P(prize|spam) = 0.2 \times 0.7 = 0.14$
$P(morning \space prize|not \space spam) = P(morning|not \space spam) \times P(prize|not \space spam) = 0.4 \times 0.1 = 0.04$
$P(morning \space prize) = 0.14 + 0.04 = 0.18$

Step 4 - Posterior - Find probability of our dependent variable given our independent variable
Now we have all the variables to solve our equation. $$P(spam|morning \space prize) = \frac{P(morning \space prize|spam) \times P(spam)}{P(morning \space prize)}$$
$P(spam|morning \space prize) = \frac{0.14 \times 0.5}{0.18} = 0.3\dot{8} \approx 0.39$
$P(not \space spam|morning \space prize) = \frac{0.04 \times 0.5}{0.18} = 0.1428.. \approx 0.14$

Hence we find around 39% chance this email is a spam and 14% chance it is not spam. If this email was in our model we would classify it as spam.

Assumptions¶

In a Naive Bayes model we presume that words are independent of one another, so we don't look at the order of words. This is where the algorithm's name "Naive" comes from.

We will use a Gaussian Naive Bayes Algorithm to build our prediction model. It is a model that can outperform other models when we don't have a lot of training data, it also runs quickly so we don't need a lot of time to train it.

Real world examples of where Naive Bayes models are used:

  • Marking email as spam or not spam
  • Classifying articles. E.g. Sports, technology, lifestyle
  • Sentiment analysis. E.g. Is this product review positive or negative?

TP = True Positive
TN = True Negative
FP = False Positive
FN = False Negative

Accuracy $= \frac{TP+TN}{TP+FP+FN+TN}$

Accuracy is the number of correct predictions over total predictions. This can be a good metric if we have symmetrical data, that is, equal amount of positive and negative values in the dataset. If we have asymmetrical data we may have a model that returns high accuracy, but fail to predict any of the minority case.

Precision $= \frac{TP}{TP+FP}$

Precision aka Specificity is the ratio of correct positive predictions over the total positive predictions. How few False Positives did we find?
Precision = 1 means no False Positives found. We predicted positive scenarios perfectly.
Pricision = 0 means all False Positives were found. We predicted no positive scenarios.

Recall $= \frac{TP}{TP+FN}$

Recall aka Sensitivity is the ratio of correct positive predictions over the total actual positive values. How few False Negatives did we find?
Recall = 1 means no False Negatives found. We never predicted a Positive value to be Negative.
Recall = 0 means all False Negatives were found. We predicted all Positive values to be Negative.

There is a natural trade off between precision and recall. If we wanted perfect recall, we can simply predict all data is Positive. We would have many False Positives (FP) but no False Negatives (FN) hence Recall = 1. Knowing this we can use the F1-Score as a balanced metric.

F1-Score $= \frac{2\times(Recall\times Precision)}{Recall + Precision}$

F1-Score is the weighted average of Precision and Recall.
