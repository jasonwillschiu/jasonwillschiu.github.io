Predictive Fishing?¶

The purpose of this project is two fold.

  1. To create a usable system to help decide where to fish on any given day
  2. To learn and document how to use data tools

Data Tools¶

  • Python. Pandas, Sklearn, APIs and later Scrapy. Will use these to web scrape, perform Extract Transform Load (ETL) and conduct analysis with Machine Learning.
  • SQL. AWS RDS instance with PostgreSQL. In part 2, we'll load our data into SQL and perform ETL.
  • Tableau. In part 3, we'll create the front-end dashboard. Users will have an interactive dashboard that will feed from our SQL back-end. Filters can be used to predict where to fish based on weather forecasts.

The Value In Predictive Fishing¶

It's on my mind almost every time I go fishing, did I pick the right spot? This project aims to suggest fishing spots based on weather conditions and previous catches.

The algorithm will consider safety and catch rate, then suggest a spot to fish. Along with the spot will be information on what fish are commonly caught there and the baits/lures that caught them.

A user will interact with the dashboard where they can view suggested fishing spots for the next few days or run custom filters to test theories.

Limitations¶

The main limitation at this point is that the dataset is my personal fishing journal. The order of magnitude is hundreds of entries, so the Machine Learning models we build later will have high variance. To fix this problem, when I have an Minimum Viable Product I plan to crowd source the dataset and the models should improve in prediction from there.

Another problem is hidden variables. We recognise that there will be factors that are hidden and that we don't plan to incorporate in our model. One example is food. Fish eat each other, if we knew where the food is, we'll know where we can catch the bigger fish. Since we don't plan on tracking little (bait) fish, we'll hopefully find that a feature such as location (Spot) and weather conditions become a proxy for food.

How scalable is Google Sheets?¶

Google Sheets is my spreadsheet of choice. I frequently use it to compare (perhaps trivial) things like booking flights or which laptop to buy. It can definitely be used for a lot more, but how much more?

I looked into how much data Google sheets can handle?. The answer is 2 million cells. Which could have the following shapes:

  • rows x cols = 2,000,000 x 1 (column A only)
  • rows x cols = 76,923 x 26 (columns A-Z)

In case you were curious this is an imposed limit, you can't create more than 2 million cells. As such Google Sheets won't be an option if you have a large/growing dataset. I can imagine some fun little apps being built with a Google Sheets backend, so it's good to keep our limitations in mind.

This is one reason that we'll move the data to SQL in part 2.

Enabling Google APIs¶

I followed this guide from the Twilio blog. See the step-by-step guide with the handy animation. A high level overview of what is required:

  1. Enable Google Sheets and Google Drive APIs via the Google API Console
  2. Download the credentials key as a JSON file. You can think of this as a password file, so Google Sheets knows it is you (and not a stranger) who is accessing your sheets. Name this file 'client_secret.json' or another name you like.
  3. Make sure your sheet is shared with at least the special Google generated email address. Mine starts with jason-787@brilliant-tide-. You can also share your sheet with everyone if you don't want to manually add email addresses.

Once this is setup we can move onto the Python setup required.

# Finds all unique types with count in a dataframe per column
def findtypes(df,series=False):
    d = {}
    if series == False:
        for item in df:
            clas = re.findall('\'(.*?)\'',(str(pd.Series(list(map(type,df[item]))).unique())))
            vals = list(pd.Series(list(map(type,df[item]))).value_counts())
            d[item] = ['{}: {}'.format(clas,vals) for clas,vals in zip(clas,vals)]
        return d
    else:
        clas = re.findall('\'(.*?)\'',(str(pd.Series(list(map(type,df))).unique())))
        vals = list(pd.Series(list(map(type,df))).value_counts())
        return ['{}: {}'.format(clas,vals) for clas,vals in zip(clas,vals)]

Data Dictionary¶

I'm planning to expand many of these variables/features in future work.

Defining a unique entry¶

A unique entry (row) is a superset of Date, Spot, Bait/Lures and Fish Caught. All other variables are dependent on Date and Spot.

DateDay Month and Year. Fish can be seasonal with the change in air/water temperature
SpotThe alias/name of a location we are fishing. Before any data analysis, this should be the most important factor in dictating which fish can be caught.
Baits/LuresWhat we used to tempt the fish to bite? Should also be an important factor
Fish CaughtThe number of fish caught and the species. I will later divide this into subsets of legal and undersize fish
Water TempSurface temperature of the water. Certain fish travel on warm/cool currents so this should dictate when certain species will likely show up.
SwellHow big the waves are, typically measured in meters or feet. Specifically the magnitude, direction and period of the swell. This is the primary factor that influences safety.
WindWind speed and direction. A factor that influences safety, comfort and how practical the fishing will be. Generally we will bare with a moderate wind but hide from a strong winds.
Time fishedStart and finish time. There could be patterns in when fish are biting.
In what tideMaximum and minimum amount of water in a certain place based on the Sun, Moon and Earth's rotation. Features of importance are tide direction (incoming or outgoing?), rate of change of water and the maximum and minimum amount of water. Different spots fish better in different tides and certain fish seem to bite in certain tides.
Moon phaseHow much moon is showing as a percentage. The moon like the sun affects the tides and the light the moon gives affects feeding patterns in some fish.
RainIf it rained or not. One theory is that fish will be hungry before and during a storm, but a couple of days after they will not bite.
Sun / CloudIf it was sunny, cloudy or both that day. Some fish bite more aggressively with less sunlight. Since I only like to fish in the day, cloudy days could be good. Some fish bite best at night.
NotesSubjective comments

# Takes the last non-iffy value and copies it to all preceding iffys 
# until the next non-iffy
def fill_down_if(series,iffy):
    latest_content = ''
    output = []
    for item in series:
        if item != iffy:
            latest_content = item
        elif item == iffy:
            item = latest_content
        output.append(item)
    return output
